PHP

I'm astonished to discover that for quite a long time, the Zend engine compiled the script to opcode EVERY SINGLE TIME a reqeust comes in...Geez! And the opcache(caching the opcode) was not bundled to it by default until PHP5.5(around 2013). This seemed to lead to some peculiar practices...

And uh...It seems that the crappy code base of Zend is making it really hard to get the code JITted. No wonder Facebook started HHVM...
